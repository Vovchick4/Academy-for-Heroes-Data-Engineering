# Pandas для Data Engineering — Теорія

## 1. Що таке Pandas?

**Pandas** — бібліотека Python для аналізу та маніпуляції табличними даними.
Побудована на NumPy, надає дві основні структури: **Series** (1D) та **DataFrame** (2D).

У Data Engineering Pandas використовується як основний двигун трансформацій в ETL/ELT-пайплайнах.

## 2. Основні структури

| Структура | Опис | Аналогія |
|-----------|------|----------|
| **Series** | Одновимірний масив з індексом | Стовпець таблиці |
| **DataFrame** | Двовимірна таблиця з колонками та індексом | SQL-таблиця / Excel-лист |

### Ключові атрибути DataFrame:
- `df.shape` — розмірність (рядки, колонки)
- `df.dtypes` — типи даних кожної колонки
- `df.columns` — назви колонок
- `df.index` — індекс (row labels)
- `df.values` — дані як numpy array
- `df.info()` — зведена інформація про DataFrame

## 3. Індексація та доступ до даних

| Метод | Опис | Приклад |
|-------|------|---------|
| `df['col']` | Доступ до стовпця | `df['name']` |
| `df[['a', 'b']]` | Декілька стовпців | `df[['name', 'salary']]` |
| `df.iloc[]` | Доступ за числовим індексом | `df.iloc[0:5, 0:2]` |
| `df.loc[]` | Доступ за міткою/умовою | `df.loc[df['age'] > 30, ['name']]` |
| `df.at[]` | Швидкий доступ до скаляра | `df.at[0, 'name']` |
| `df.iat[]` | Швидкий доступ за координатами | `df.iat[0, 1]` |

## 4. Читання та запис даних (I/O)

| Формат | Читання | Запис | Коли використовувати |
|--------|---------|-------|---------------------|
| **CSV** | `pd.read_csv()` | `df.to_csv()` | Універсальний формат обміну |
| **Excel** | `pd.read_excel()` | `df.to_excel()` | Звіти, бізнес-дані |
| **JSON** | `pd.read_json()` | `df.to_json()` | API, веб-дані |
| **Parquet** | `pd.read_parquet()` | `df.to_parquet()` | Великі дані, Data Lake (стиснення + швидкість) |
| **SQL** | `pd.read_sql()` | `df.to_sql()` | Бази даних, DWH |
| **Feather** | `pd.read_feather()` | `df.to_feather()` | Швидкий обмін між мовами |

### Поради для великих файлів:
- `chunksize=N` — читання частинами (ітератор)
- `usecols=[...]` — читати тільки потрібні колонки
- `dtype={'col': type}` — задати типи при читанні
- `nrows=N` — прочитати N рядків для розвідки
- `low_memory=False` — уникнути mixed type warnings

## 5. Розвідка даних (EDA)

Обов'язкові кроки перед будь-якою трансформацією:

```python
df.head(10)          # перші рядки
df.info()            # типи + non-null count
df.describe()        # статистика (числові колонки)
df.describe(include='all')  # статистика + категоріальні
df.isnull().sum()    # кількість пропусків
df.duplicated().sum() # дублікати
df['col'].value_counts()  # розподіл значень
df['col'].nunique()  # кількість унікальних
```

## 6. Очищення даних (Data Cleaning)

### Пропуски (NaN/None)
| Метод | Опис |
|-------|------|
| `dropna()` | Видалити рядки/стовпці з NaN |
| `fillna(value)` | Заповнити значенням |
| `fillna(method='ffill')` | Forward fill (попереднім значенням) |
| `fillna(method='bfill')` | Backward fill (наступним значенням) |
| `interpolate()` | Інтерполяція (лінійна, часова) |

### Дублікати
- `df.duplicated()` — виявити дублікати
- `df.drop_duplicates()` — видалити дублікати
- `df.drop_duplicates(subset=['col1', 'col2'], keep='last')`

### Типи даних
- `df['col'].astype(type)` — конвертація типу
- `pd.to_datetime()` — парсинг дат
- `pd.to_numeric()` — конвертація в числа (errors='coerce')
- Категоріальний тип `'category'` для економії пам'яті

### Викиди (Outliers)
- **IQR метод**: Q1 - 1.5*IQR ... Q3 + 1.5*IQR
- **Z-score**: |z| > 3 — викид
- **Capping**: `df['col'].clip(lower, upper)`

## 7. Фільтрація та сортування

```python
# Boolean indexing
df[(df['dept'] == 'IT') & (df['salary'] > 50000)]

# query() — SQL-подібний синтаксис
df.query('dept == "IT" and salary > 50000')

# isin() — фільтр за списком
df[df['dept'].isin(['IT', 'Finance'])]

# between() — діапазон
df[df['age'].between(25, 35)]

# Сортування
df.sort_values('salary', ascending=False)
df.sort_values(['dept', 'salary'], ascending=[True, False])
```

## 8. GroupBy та Агрегація

```python
# Одна агрегація
df.groupby('dept')['salary'].mean()

# Декілька агрегацій
df.groupby('dept')['salary'].agg(['mean', 'median', 'std', 'count'])

# Різні агрегації для різних колонок
df.groupby('dept').agg({
    'salary': 'mean',
    'age': ['min', 'max'],
    'user_id': 'count'
})

# Named aggregation (чистий синтаксис)
df.groupby('dept', as_index=False).agg(
    avg_salary=('salary', 'mean'),
    emp_count=('user_id', 'count')
)

# transform — агрегація для кожного рядка
df['dept_avg'] = df.groupby('dept')['salary'].transform('mean')

# pivot_table
pd.pivot_table(df, values='salary', index='dept', columns='is_active', aggfunc='mean')
```

## 9. Об'єднання даних

| Функція | Тип | Опис |
|---------|-----|------|
| `pd.merge()` | SQL-подібний | Об'єднання за ключами (inner/left/right/outer) |
| `pd.concat()` | Склеювання | Вертикальне/горизонтальне об'єднання |
| `df.join()` | На індексі | Об'єднання за індексом |

### Merge — типи join:
- `how='inner'` — тільки співпадіння (за замовчуванням)
- `how='left'` — всі з лівого + співпадіння з правого
- `how='right'` — всі з правого + співпадіння з лівого
- `how='outer'` — всі рядки з обох таблиць

## 10. Робота з датами

```python
pd.to_datetime()           # парсинг
df['date'].dt.year         # рік
df['date'].dt.month        # місяць
df['date'].dt.day          # день
df['date'].dt.dayofweek    # день тижня (0=Monday)
df['date'].dt.quarter      # квартал
df['date'].dt.isocalendar().week  # номер тижня

# Ресемплінг
df.set_index('date').resample('ME').mean()  # місячний
df.set_index('date').resample('W').sum()    # тижневий

# Rolling window
df['rolling_avg'] = df['value'].rolling(window=7).mean()

# Різниця в датах
(df['date2'] - df['date1']).dt.days
```

## 11. Оптимізація пам'яті та продуктивності

1. **Правильні типи**: float64 → float32, int64 → int8/16/32
2. **Категоріальний тип**: object → category (економія 90%+)
3. **Векторизація замість apply()**: в 10-100x швидше
4. **Parquet замість CSV**: в 2-5x швидше, менше місця
5. **Chunked reading**: обробка частинами для файлів > RAM
6. **query()** трохи швидше за boolean indexing
7. **inplace=False** (явне присвоєння) — краще за inplace=True

## 12. Polars vs Pandas

| Критерій | Pandas | Polars |
|----------|--------|--------|
| **Ядро** | Python/C | Rust |
| **Ліниві обчислення** | Ні | Так (LazyFrame) |
| **Паралельність** | 1 ядро (default) | Всі ядра CPU |
| **Індекс** | Є (важливий) | Немає |
| **Expression API** | Обмежений | Повноцінний |
| **SQL support** | Через `pd.read_sql` | Нативний `.sql()` |
| **Коли використовувати** | < RAM, ML, візуалізація | > RAM, продуктивність |

> **Стратегія DE**: починайте з Pandas, мігруйте на Polars коли дані стають "великими" або продуктивність критична.

## 13. Типовий ETL-пайплайн на Pandas

```
Extract:  pd.read_csv / pd.read_sql / pd.read_parquet
          ↓
Transform:  чищення → фільтрація → трансформація → агрегація
          ↓
Load:    df.to_sql / df.to_parquet / df.to_csv
```

### Шаблон:

```python
def run_etl(source_path: str, dest_path: str) -> None:
    df = pd.read_csv(source_path)
    df = df.dropna(subset=['key_column'])
    df['date'] = pd.to_datetime(df['date'])
    df = df[df['status'] == 'active']
    result = df.groupby('category').agg({'value': 'sum'}).reset_index()
    result.to_parquet(dest_path, index=False)
```
