# Вступ до PySpark — Теорія

## 1. Що таке Apache Spark?

**Apache Spark** — фреймворк для розподіленої обробки даних (distributed computing). Працює в пам'яті (in-memory), що робить його значно швидшим за Hadoop MapReduce.

**PySpark** — офіційний Python API для Spark.

## 2. Ключові концепції

| Поняття | Опис |
|---------|------|
| **RDD** | Resilient Distributed Dataset — низькорівневе API (не використовується напряму) |
| **DataFrame** | Основне API (колонки + schema). Як Pandas, але розподілено |
| **SparkSession** | Вхідна точка для роботи з Spark |
| **Transformation** | Лінива операція (describe, filter, select, groupBy) |
| **Action** | Запускає обчислення (show, count, write, collect) |
| **DAG** | Directed Acyclic Graph — план виконання, який оптимізує Catalyst |
| **Cluster** | Набір машин (driver + executors) |

## 3. Spark vs Pandas

| Критерій | Pandas | PySpark |
|----------|--------|---------|
| Обсяг даних | < RAM (1-10 ГБ) | >> RAM (ТБ-ПБ) |
| Обробка | Одне ядро | Розподілено (кластер) |
| Evaluation | Eager (одразу) | Lazy (при action) |
| Індекс | Є | Немає |
| SQL | Через read_sql | Нативний Spark SQL |
| Коли використовувати | Прототипи, малі дані, ML | Продакшн, великі дані, ETL |

## 4. DataFrame API

```python
# Створення
df = spark.createDataFrame(data, schema=['col1', 'col2'])

# Читання
df = spark.read.csv('file.csv', header=True, inferSchema=True)
df = spark.read.parquet('file.parquet')
df = spark.read.json('file.json')

# Операції
df.select('col1', 'col2')
df.filter(col('salary') > 50000)
df.groupBy('dept').agg(avg('salary').alias('avg_sal'))
df.withColumn('new_col', expr('old_col * 2'))
df.sort(col('salary').desc())
df.limit(10)

# Екшени
df.show()        # вивести N рядків
df.count()       # кількість рядків
df.printSchema() # схема
df.collect()     # зібрати всі дані в драйвер (обережно!)
df.write.parquet('output/')  # запис
```

## 5. Spark SQL

```python
df.createOrReplaceTempView('my_table')
result = spark.sql('SELECT * FROM my_table WHERE col > 10')
```

## 6. Формати даних

| Формат | Читання | Запис | Коли використовувати |
|--------|---------|-------|---------------------|
| **Parquet** | `spark.read.parquet()` | `df.write.parquet()` | ⭐ Основний формат для Spark |
| **Delta Lake** | `spark.read.format('delta')` | `df.write.format('delta')` | Транзакції, time travel |
| **CSV** | `spark.read.csv()` | `df.write.csv()` | Обмін даними |
| **JSON** | `spark.read.json()` | `df.write.json()` | API, документи |
| **JDBC** | `spark.read.format('jdbc').option(...)` | `df.write.format('jdbc')` | Бази даних |

## 7. Коли використовувати Spark?

**Так:** дані > 100 ГБ, ETL на кластері, Data Lake, Streaming, терабайти на день

**Ні:** дані < 10 ГБ, прототипування, візуалізація, ad-hoc аналіз

## 8. Типовий DE-патерн на Spark

```python
# ETL на Spark
df = spark.read.parquet('/data/landing/orders')
df_clean = df.filter(col('status') == 'active')
df_result = df_clean.groupBy('category').agg(sum('amount').alias('total'))
df_result.write.mode('overwrite').parquet('/data/dwh/orders_by_category')
```
