# 📘 SQL Interview Tasks: Functions & Stored Procedures

# 🧩 Задача 1 — Scalar Function (бізнес-логіка)

## 📊 Таблиця `products`

| id | name  | price |
| -- | ----- | ----- |
| 1  | Phone | 1000  |
| 2  | TV    | 2000  |
| 3  | Book  | 100   |

---

## ❓ Завдання

Створити функцію, яка:

* застосовує знижку:

  * > 1000 → 10%
  * ≤ 1000 → 5%
* повертає нову ціну

---

## ✅ Рішення

```sql
CREATE FUNCTION get_discount_price(price DECIMAL)
RETURNS DECIMAL
AS
BEGIN
    RETURN CASE 
        WHEN price > 1000 THEN price * 0.9
        ELSE price * 0.95
    END;
END;
```

---

## ✅ Використання

```sql
SELECT name, get_discount_price(price) AS final_price
FROM products;
```

---

## 💡 Підступ

👉 функція не повинна змінювати дані
👉 тільки обчислення

# 🧩 Задача 2 — Table-valued Function

## 📊 Таблиця `orders`

| id | user_id | amount |
| -- | ------- | ------ |
| 1  | 1       | 500    |
| 2  | 1       | 700    |
| 3  | 2       | 200    |

---

## ❓ Завдання

Створити функцію, яка повертає:

* всі замовлення користувача
* по `user_id`

---

## ✅ Рішення

```sql
CREATE FUNCTION get_user_orders(u_id INT)
RETURNS TABLE
AS
RETURN (
    SELECT *
    FROM orders
    WHERE user_id = u_id
);
```

---

## ✅ Використання

```sql
SELECT * FROM get_user_orders(1);
```

---

## 💡 Підступ

👉 перевіряють чи знаєш різницю:

* scalar vs table function

# 🧩 Задача 3 — Function у WHERE (performance trap)

## ❓ Завдання

Знайти продукти зі знижкою > 900

---

## ❌ Погане рішення

```sql
SELECT *
FROM products
WHERE get_discount_price(price) > 900;
```

---

## ⚠️ Проблема

* функція викликається для кожного рядка
* індекси НЕ працюють

---

## ✅ Краще рішення

```sql
SELECT *
FROM products
WHERE price > 1000; -- логіку переносимо сюди
```

---

## 💡 Підступ

👉 senior рівень = думає про performance

# 🧩 Задача 4 — Stored Procedure (створення замовлення)

## 📊 Таблиці

### `users`

| id | balance |
| -- | ------- |
| 1  | 1000    |

### `orders`

| id | user_id | amount |
| -- | ------- | ------ |

---

## ❓ Завдання

Створити процедуру:

* створює замовлення
* списує баланс
* якщо грошей нема → rollback

---

## ✅ Рішення

```sql
CREATE PROCEDURE create_order(
    IN u_id INT,
    IN order_amount DECIMAL
)
AS
BEGIN
    DECLARE user_balance DECIMAL;

    SELECT balance INTO user_balance
    FROM users
    WHERE id = u_id;

    IF user_balance < order_amount THEN
        SIGNAL SQLSTATE '45000'
        SET MESSAGE_TEXT = 'Not enough balance';
    END IF;

    START TRANSACTION;

    INSERT INTO orders(user_id, amount)
    VALUES (u_id, order_amount);

    UPDATE users
    SET balance = balance - order_amount
    WHERE id = u_id;

    COMMIT;
END;
```

---

## 💡 Підступ

👉 перевіряють:

* транзакції
* помилки
* бізнес-логіку

# 🧩 Задача 5 — Procedure з OUT параметром

## ❓ Завдання

Отримати баланс користувача через процедуру

---

## ✅ Рішення

```sql
CREATE PROCEDURE get_user_balance(
    IN u_id INT,
    OUT user_balance DECIMAL
)
AS
BEGIN
    SELECT balance INTO user_balance
    FROM users
    WHERE id = u_id;
END;
```

---

## ✅ Виклик

```sql
CALL get_user_balance(1, @balance);
SELECT @balance;
```

---

## 💡 Підступ

👉 OUT параметри часто питають

# 🧩 Задача 6 — Function vs Procedure (що вибрати?)

## ❓ Завдання

Потрібно:

* порахувати суму замовлення
* і використати її в SELECT

---

## ✅ Правильна відповідь

👉 Function

```sql
CREATE FUNCTION get_order_total(order_id INT)
RETURNS DECIMAL
AS
BEGIN
    RETURN (
        SELECT SUM(amount)
        FROM orders
        WHERE id = order_id
    );
END;
```

---

## ❌ Чому не Procedure?

👉 бо:

* не можна використати в SELECT
* не повертає значення напряму

# 🧩 Задача 8 — Реальний кейс (e-commerce)

## ❓ Завдання

Реалізувати:

### 1. Function:

* рахує суму замовлення

```sql
CREATE FUNCTION calc_order_total(o_id INT)
RETURNS DECIMAL
AS
BEGIN
    RETURN (
        SELECT SUM(price)
        FROM order_items
        WHERE order_id = o_id
    );
END;
```

---

### 2. Procedure:

* оформляє замовлення

```sql
CREATE PROCEDURE checkout(
    IN u_id INT,
    IN o_id INT
)
AS
BEGIN
    DECLARE total DECIMAL;

    SET total = calc_order_total(o_id);

    START TRANSACTION;

    UPDATE users
    SET balance = balance - total
    WHERE id = u_id;

    UPDATE orders
    SET status = 'paid'
    WHERE id = o_id;

    COMMIT;
END;
```

---

## 💡 Підступ

👉 комбінування:

* function + procedure

---

# 🔥 Що реально перевіряють

На співбесіді дивляться чи ти розумієш:

* де function, де procedure
* side effects*
* транзакції
* performance
* reusable логіку
* обмеження функцій

*🔹 Що це таке

Side effects (побічні ефекти) — це будь-які зміни стану системи, які відбуваються під час виконання функції або процедури.
