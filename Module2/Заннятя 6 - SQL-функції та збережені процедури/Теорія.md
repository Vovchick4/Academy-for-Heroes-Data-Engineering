# 📘 УРОК: SQL-функції та збережені процедури

---

# 🔹 1. Що це взагалі таке

У БД є два способи інкапсулювати логіку:

* **Функції (Functions)** → повертають значення
* **Збережені процедури (Stored Procedures)** → виконують дії

👉 Обидва — це **збережений код в БД**, як “бекенд всередині БД”

---

# 🔹 2. SQL-функції (Functions)

## 📌 Визначення

Функція — це об’єкт БД, який:

* приймає параметри
* **завжди повертає значення**
* може використовуватись у SQL-запитах

---

## 🔸 Типи функцій

### 1. Scalar (скалярні)

Повертають 1 значення

```sql
CREATE FUNCTION get_tax(price DECIMAL)
RETURNS DECIMAL
AS
BEGIN
    RETURN price * 0.2;
END;
```

---

### 2. Table-valued (табличні)

Повертають таблицю

```sql
CREATE FUNCTION get_rich_users()
RETURNS TABLE
AS
RETURN (
    SELECT * FROM users WHERE balance > 10000
);
```

Використання:

```sql
SELECT * FROM get_rich_users();
```

---

## 🔸 Використання

```sql
SELECT name, get_tax(price)
FROM products;
```

або:

```sql
SELECT *
FROM products
WHERE price > get_avg_price();
```

---

## 🔸 Обмеження функцій

⚠️ Важливо (часто питають):

* не можна робити:

  * `INSERT`, `UPDATE`, `DELETE` (у більшості БД або обмежено)
* не можна управляти транзакціями
* не можна змінювати стан БД

👉 Функція = **чиста логіка / обчислення**

---

# 🔹 3. Stored Procedures (збережені процедури)

## 📌 Визначення

Процедура — це:

* набір SQL-операцій
* виконує дію
* може НЕ повертати значення

---

## 🔸 Приклад

```sql
CREATE PROCEDURE transfer_money(
    from_user INT,
    to_user INT,
    amount DECIMAL
)
AS
BEGIN
    UPDATE users SET balance = balance - amount WHERE id = from_user;
    UPDATE users SET balance = balance + amount WHERE id = to_user;
END;
```

---

## 🔸 Виклик

```sql
CALL transfer_money(1, 2, 500);
```

---

## 🔸 Процедури можуть:

✅ Робити:

* INSERT / UPDATE / DELETE
* транзакції
* IF / LOOP / CASE
* виклик інших процедур

---

## 🔸 Транзакції (дуже важливо 🔥)

```sql
CREATE PROCEDURE safe_transfer(...)
AS
BEGIN
    BEGIN TRANSACTION;

    UPDATE users SET balance = balance - 100 WHERE id = 1;

    IF @@ERROR <> 0
    BEGIN
        ROLLBACK;
        RETURN;
    END;

    UPDATE users SET balance = balance + 100 WHERE id = 2;

    COMMIT;
END;
```

👉 Це класичний приклад для співбесіди

---

# 🔥 4. Ключові відмінності

| Критерій              | Function      | Procedure        |
| --------------------- | ------------- | ---------------- |
| Повертає значення     | ✅ обов’язково | ❌ не обов’язково |
| Використання в SELECT | ✅             | ❌                |
| Модифікація БД        | ❌             | ✅                |
| Транзакції            | ❌             | ✅                |
| Виклик                | в запиті      | CALL / EXEC      |
| Призначення           | обчислення    | бізнес-логіка    |

---

# 🔥 5. Коли що використовувати

## ✅ Використовуй FUNCTION коли:

* потрібно обчислення
* логіка reusable
* потрібно в SELECT / WHERE

👉 приклади:

* розрахунок податку
* знижка
* форматування даних

---

## ✅ Використовуй PROCEDURE коли:

* складна логіка
* робота з кількома таблицями
* потрібні транзакції

👉 приклади:

* створення замовлення
* переказ грошей
* обробка платежів

---

# 🔥 6. Реальний приклад (як у продакшені)

## 🛒 Інтернет-магазин

### 🔹 Функція

```sql
get_order_total(order_id)
```

👉 рахує суму замовлення

---

### 🔹 Процедура

```sql
create_order(user_id, items)
```

👉 робить:

* створює order
* додає items
* списує баланс
* логування

---

# 🔥 7. Advanced теми (що питають на senior)

## 🔸 1. Deterministic vs Non-deterministic

* **Deterministic** → завжди один результат
* **Non-deterministic** → різний результат (наприклад `NOW()`)

---

## 🔸 2. Performance

⚠️ Важливо:

* функції можуть **погіршувати performance**
* особливо у `WHERE` і `JOIN`

👉 приклад погано:

```sql
WHERE get_discount(price) > 100
```

---

## 🔸 3. Security

✅ Procedures можна використовувати для:

* обмеження доступу до таблиць
* доступ тільки через процедуру

---

## 🔸 4. Side effects

* функції → без side effects
* процедури → мають side effects

---

## 🔸 5. Повернення даних з procedure

```sql
SELECT * FROM users;
```

або через OUT параметри:

```sql
CREATE PROCEDURE get_balance(IN user_id INT, OUT balance DECIMAL)
```

---

# 🔥 8. Коли НЕ використовувати

👉 У сучасних проектах часто логіку виносять у:

* Node.js
* Python
* Java

Чому:

* легше тестувати
* легше масштабувати
* контроль версій

---

# 🔥 9. Як відповісти на співбесіді (готова відповідь)

> SQL-функції використовуються для обчислень і завжди повертають значення, тому можуть бути частиною SELECT-запитів. Збережені процедури інкапсулюють бізнес-логіку, можуть виконувати складні операції над даними, включаючи транзакції, і викликаються окремо. У продакшені функції застосовують для reusable обчислень, а процедури — для операцій типу створення замовлення або фінансових транзакцій.
