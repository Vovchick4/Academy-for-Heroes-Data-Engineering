# 📘 УРОК: SQL-функції та збережені процедури (з урахуванням реальності MySQL)

---

# 🔹 1. Що це таке

У БД є 2 способи писати логіку:

* **Function** → повертає значення (як `return`)
* **Procedure** → виконує дії (як “скрипт”)

👉 Це як backend, але прямо в БД

---

# 🔹 2. SQL-функції (Function)

## 📌 Визначення

Функція:

* приймає параметри
* **обов’язково повертає значення**
* може використовуватись у `SELECT`, `WHERE`

---

# 🔥 3. ДВА СПОСОБИ створення функцій у MySQL

І це найважливіше, що ти вже відчув 😄

---

## ✅ ВАРІАНТ 1 — Простий (працює ВСЮДИ)

```sql
CREATE FUNCTION get_tax(price DECIMAL(10,2))
RETURNS DECIMAL(10,2)
DETERMINISTIC
RETURN price * 0.2;
```

### ✔ Плюси:

* працює в:

  * DBeaver
  * DataGrip
  * phpMyAdmin
* не потрібен `DELIMITER`
* ідеально для простих задач

### ❌ Мінуси:

* тільки 1 вираз (без IF, LOOP)

---

## ❌ ВАРІАНТ 2 — Через BEGIN (часто ЛАМАЄТЬСЯ)

```sql
DELIMITER $$

CREATE FUNCTION get_tax(price DECIMAL(10,2))
RETURNS DECIMAL(10,2)
DETERMINISTIC
BEGIN
    RETURN price * 0.2;
END $$

DELIMITER ;
```

---

## 💥 Чому не працює

👉 Більшість SQL-клієнтів **НЕ підтримують `DELIMITER`**

Що відбувається:

```sql
BEGIN
    RETURN price * 0.2;
END;
```

👉 клієнт бачить `;` і думає:

> "запит завершено"

❌ функція не створюється
❌ помилки можуть не показатись

---

## 🔥 Висновок

| Ситуація            | Що використовувати  |
| ------------------- | ------------------- |
| проста логіка       | ✅ коротка функція   |
| складна логіка      | ⚠️ тільки через CLI |
| IDE (DBeaver і т.д) | ❌ без BEGIN         |

---

# 🔹 4. Коли потрібен BEGIN

Коли є логіка:

```sql
IF price > 1000 THEN ...
```

Приклад:

```sql
DELIMITER $$

CREATE FUNCTION get_discount(price DECIMAL(10,2))
RETURNS DECIMAL(10,2)
DETERMINISTIC
BEGIN
    IF price > 1000 THEN
        RETURN price * 0.9;
    ELSE
        RETURN price;
    END IF;
END $$

DELIMITER ;
```

👉 це **НЕ запрацює в більшості IDE**

---

# 🔹 5. Як перевірити чи функція існує

## ✅ Варіант 1

```sql
SHOW FUNCTION STATUS 
WHERE Name = 'get_tax';
```

---

## ✅ Варіант 2 (надійніший)

```sql
SELECT ROUTINE_NAME, ROUTINE_SCHEMA
FROM information_schema.ROUTINES
WHERE ROUTINE_TYPE = 'FUNCTION'
AND ROUTINE_NAME = 'get_tax';
```

---

# 🔹 6. Використання функції

```sql
SELECT name, get_tax(price)
FROM products;
```

---

# 🔹 7. Обмеження функцій

⚠️ Важливо:

Функції:

* ❌ не повинні змінювати БД
* ❌ не для `INSERT / UPDATE`
* ❌ без транзакцій

👉 функція = **чисте обчислення**

---

# 🔹 8. Stored Procedures (процедури)

## 📌 Визначення

Процедура:

* виконує дію
* може нічого не повертати

---

## 🔥 ВАЖЛИВО (як і з функціями)

👉 тут ті ж самі проблеми з `DELIMITER`

---

## ✅ Приклад (через CLI)

```sql
DELIMITER $$

CREATE PROCEDURE transfer_money(
    from_user INT,
    to_user INT,
    amount DECIMAL(10,2)
)
BEGIN
    UPDATE users SET balance = balance - amount WHERE id = from_user;
    UPDATE users SET balance = balance + amount WHERE id = to_user;
END $$

DELIMITER ;
```

---

## ❌ В IDE може НЕ створитись

---

## ✅ Виклик

```sql
CALL transfer_money(1, 2, 500);
```

---

# 🔹 9. Різниця Function vs Procedure

| Критерій              | Function | Procedure          |
| --------------------- | -------- | ------------------ |
| Повертає значення     | ✅        | ❌ (не обов’язково) |
| Використання в SELECT | ✅        | ❌                  |
| Змінює БД             | ❌        | ✅                  |
| Транзакції            | ❌        | ✅                  |

---

# 🔥 10. Реальний продакшн підхід

👉 У сучасних проектах:

* складна логіка → Node.js / backend
* SQL → тільки:

  * прості функції
  * агрегації

# 🔥 11. Типові помилки (як у тебе)

### ❌ Помилка 1

```
FUNCTION does not exist
```

👉 функція не створилась

---

### ❌ Помилка 2

```
syntax error near END
```

👉 немає `DELIMITER`

---

### ❌ Помилка 3

```
нічого не відбувається
```

👉 клієнт обрізає запит

---

# 🚀 12. TL;DR (дуже важливо)

👉 Якщо ти в IDE:

✔ використовуй:

```sql
RETURN expression
```

❌ не використовуй:

```sql
BEGIN ... END
```
