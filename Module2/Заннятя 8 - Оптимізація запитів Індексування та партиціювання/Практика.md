# 🧠 ТЕСТ: Оптимізація SQL, Індекси, Партиції

---

## ✅ 1. Який запит використовує індекс?

A)

```sql
SELECT * FROM orders WHERE YEAR(created_at) = 2024;
```

B)

```sql
SELECT * FROM orders 
WHERE created_at >= '2024-01-01' 
AND created_at < '2025-01-01';
```

**👉 Відповідь: B**

**Чому:**
Функція `YEAR()` ламає індекс. У B — чисте порівняння → індекс працює.

---

## ✅ 2. Що буде без індексу?

```sql
SELECT * FROM users WHERE email = 'test@mail.com';
```

A) Binary search
B) Full Table Scan
C) Hash lookup

**👉 Відповідь: B**

**Чому:**
Без індексу SQL читає **всю таблицю**.

---

## ✅ 3. Який індекс потрібен?

```sql
SELECT * FROM orders WHERE user_id = 10;
```

A) INDEX(amount)
B) INDEX(user_id)
C) INDEX(created_at)

**👉 Відповідь: B**

---

## ✅ 4. Composite index

Є індекс:

```sql
(user_id, created_at)
```

Який запит використає індекс?

A)

```sql
WHERE created_at > '2024-01-01'
```

B)

```sql
WHERE user_id = 1
```

C)

```sql
WHERE created_at > '2024' AND user_id = 1
```

**👉 Відповідь: B і C**

**Чому:**
Індекс працює **зліва направо**.

---

## ✅ 5. LIKE і індекс

Який варіант використовує індекс?

A)

```sql
WHERE name LIKE '%ivan%'
```

B)

```sql
WHERE name LIKE 'ivan%'
```

**👉 Відповідь: B**

**Чому:**
`%` на початку → індекс не працює.

---

## ✅ 6. Що покаже EXPLAIN?

```sql
EXPLAIN SELECT * FROM orders WHERE user_id = 5;
```

A) Як написаний SQL
B) План виконання
C) Дані таблиці

**👉 Відповідь: B**

---

## ✅ 7. JOIN оптимізація

```sql
SELECT *
FROM orders o
JOIN users u ON o.user_id = u.id;
```

Що потрібно?

A) INDEX(users.name)
B) INDEX(orders.user_id)
C) INDEX(orders.amount)

**👉 Відповідь: B**

---

## ✅ 8. Коли індекс НЕ допоможе?

A) Таблиця 100 рядків
B) Таблиця 10 млн рядків
C) Частий WHERE

**👉 Відповідь: A**

**Чому:**
Маленькі таблиці → швидше без індексу.

---

## ✅ 9. Partitioning

Що дає partition?

A) Швидший INSERT
B) Менше даних для читання
C) Менше пам’яті

**👉 Відповідь: B**

---

## ✅ 10. Оптимізація ORDER BY

```sql
SELECT * FROM orders
WHERE user_id = 5
ORDER BY created_at DESC;
```

Який індекс?

A)

```sql
(user_id)
```

B)

```sql
(created_at)
```

C)

```sql
(user_id, created_at)
```

**👉 Відповідь: C**

# ✅ РІШЕННЯ

## 1. Індекс

```sql
CREATE INDEX idx_status_date 
ON orders(status, created_at DESC);
```

---

## 2. Partition

👉 Так, якщо таблиця велика

```sql
PARTITION BY RANGE (YEAR(created_at))
```

---

## 3. Додаткова оптимізація

* не використовувати `SELECT *`
* брати тільки потрібні поля:

```sql
SELECT id, user_id, amount
FROM orders
...
```

# 💡 Висновок

Щоб писати швидкий SQL:

* думай як **база читає дані**
* використовуй **індекси**
* перевіряй через **EXPLAIN**
* оптимізуй **WHERE + ORDER BY**
* використовуй **partition для великих таблиць**
