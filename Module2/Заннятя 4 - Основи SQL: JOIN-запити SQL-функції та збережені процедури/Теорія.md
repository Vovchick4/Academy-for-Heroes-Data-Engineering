# Частина 1 — Агрегація даних

(30–40 хв)

У реальному Data Engineering **90% SQL запитів — це агрегування**.

Агрегація — це коли ми **об'єднуємо багато рядків у один результат**.

Наприклад:

| order_id | user | price |
| -------- | ---- | ----- |
| 1        | Ivan | 100   |
| 2        | Ivan | 200   |
| 3        | Anna | 300   |

Можемо порахувати:

* загальну суму
* середню ціну
* кількість замовлень

---

## Основні агрегатні функції

| Функція | Опис                 |
| ------- | -------------------- |
| COUNT() | кількість рядків     |
| SUM()   | сума                 |
| AVG()   | середнє              |
| MAX()   | максимальне значення |
| MIN()   | мінімальне значення  |

---

## Приклад таблиці

```sql
CREATE TABLE orders (
    id INT,
    user_name VARCHAR(50),
    amount INT
);
```

Дані:

```sql
INSERT INTO orders VALUES
(1,'Ivan',100),
(2,'Ivan',200),
(3,'Anna',300),
(4,'Anna',150),
(5,'Oleh',400);
```

---

## COUNT()

Порахувати кількість замовлень

```sql
SELECT COUNT(*) 
FROM orders;
```

Результат

```
5
```

---

## SUM()

Сума всіх продажів

```sql
SELECT SUM(amount)
FROM orders;
```

---

## AVG()

Середній чек

```sql
SELECT AVG(amount)
FROM orders;
```

---

# Частина 2 — GROUP BY

(40 хв)

`GROUP BY` дозволяє **групувати рядки по колонці**.

Наприклад:

Порахувати **суму продажів для кожного користувача**

```sql
SELECT user_name, SUM(amount)
FROM orders
GROUP BY user_name;
```

Результат:

| user_name | sum |
| --------- | --- |
| Ivan      | 300 |
| Anna      | 450 |
| Oleh      | 400 |

---

## Кількість замовлень на користувача

```sql
SELECT user_name, COUNT(*)
FROM orders
GROUP BY user_name;
```

---

## Середній чек

```sql
SELECT user_name, AVG(amount)
FROM orders
GROUP BY user_name;
```

---

# Частина 3 — HAVING

(20 хв)

`HAVING` використовується **для фільтрації GROUP BY**.

Важливо:

```
WHERE → фільтрує рядки
HAVING → фільтрує групи
```

---

## Приклад

Користувачі з сумою замовлень більше 400

```sql
SELECT user_name, SUM(amount)
FROM orders
GROUP BY user_name
HAVING SUM(amount) > 400;
```

Результат:

| user_name | sum |
| --------- | --- |
| Anna      | 450 |

# Частина 4 — JOIN

(50 хв)

У реальних базах **дані майже завжди розбиті на кілька таблиць**.

Наприклад:

### users

| id | name |
| -- | ---- |
| 1  | Ivan |
| 2  | Anna |
| 3  | Oleh |

---

### orders

| id | user_id | amount |
| -- | ------- | ------ |
| 1  | 1       | 100    |
| 2  | 1       | 200    |
| 3  | 2       | 300    |

---

## INNER JOIN

```sql
SELECT users.name, orders.amount
FROM users
INNER JOIN orders
ON users.id = orders.user_id;
```

Результат

| name | amount |
| ---- | ------ |
| Ivan | 100    |
| Ivan | 200    |
| Anna | 300    |

---

## LEFT JOIN

Показує **всі рядки з лівої таблиці**

```sql
SELECT users.name, orders.amount
FROM users
LEFT JOIN orders
ON users.id = orders.user_id;
```

Результат

| name | amount |
| ---- | ------ |
| Ivan | 100    |
| Ivan | 200    |
| Anna | 300    |
| Oleh | NULL   |

---

## RIGHT JOIN

```sql
SELECT users.name, orders.amount
FROM users
RIGHT JOIN orders
ON users.id = orders.user_id;
```

---

## JOIN + GROUP BY

Наприклад:

**Сума замовлень для кожного користувача**

```sql
SELECT users.name, SUM(orders.amount)
FROM users
JOIN orders
ON users.id = orders.user_id
GROUP BY users.name;
```

# Частина 5 — Пошук дублікатів

(20 хв)

Дуже часта задача у Data Engineering.

Наприклад таблиця:

| email                             |
| --------------------------------- |
| [a@gmail.com](mailto:a@gmail.com) |
| [b@gmail.com](mailto:b@gmail.com) |
| [a@gmail.com](mailto:a@gmail.com) |

---

## Знайти дублікати

```sql
SELECT email, COUNT(*)
FROM users
GROUP BY email
HAVING COUNT(*) > 1;
```

Результат:

| email                             | count |
| --------------------------------- | ----- |
| [a@gmail.com](mailto:a@gmail.com) | 2     |

# Частина 7 — Stored Procedures

(20 хв)

Stored Procedure — це **SQL код який зберігається в базі**.

Використовується для:

* складних запитів
* ETL процесів
* автоматизації

---

## Приклад

```sql
CREATE PROCEDURE get_orders()
BEGIN
    SELECT * FROM orders;
END;
```

Виклик

```sql
CALL get_orders();
```

---

## Procedure з параметром

```sql
CREATE PROCEDURE get_user_orders(IN uid INT)
BEGIN
    SELECT * 
    FROM orders
    WHERE user_id = uid;
END;
```

Виклик

```sql
CALL get_user_orders(1);
```
