**Віконні функції (Window Functions) в SQL** — це функції, які дозволяють виконувати обчислення **над набором рядків, пов’язаних з поточним рядком**, не згортаючи їх у один результат (як це робить `GROUP BY`).

Тобто вони **рахують агрегати, але залишають усі рядки**.

---

# 1. Синтаксис

```sql
FUNCTION_NAME(column)
OVER (
    PARTITION BY column
    ORDER BY column
)
```

**Частини:**

* `FUNCTION_NAME` — функція (`SUM`, `AVG`, `ROW_NUMBER` тощо)
* `OVER()` — визначає **вікно**
* `PARTITION BY` — розбиває дані на групи
* `ORDER BY` — порядок обчислення

---

# 2. Приклад таблиці

### orders

| id | user_id | amount |
| -- | ------- | ------ |
| 1  | 1       | 100    |
| 2  | 1       | 200    |
| 3  | 2       | 150    |
| 4  | 2       | 300    |
| 5  | 3       | 50     |

# 3. SUM як віконна функція

```sql
SELECT 
    id,
    user_id,
    amount,
    SUM(amount) OVER(PARTITION BY user_id) AS total_user
FROM orders;
```

Результат

| id | user_id | amount | total_user |
| -- | ------- | ------ | ---------- |
| 1  | 1       | 100    | 300        |
| 2  | 1       | 200    | 300        |
| 3  | 2       | 150    | 450        |
| 4  | 2       | 300    | 450        |
| 5  | 3       | 50     | 50         |

Тут:

```
PARTITION BY user_id
```

означає:

**рахувати суму для кожного користувача**

Ось **коротша теорія**, яку можна додати до цього прикладу.

---

### Що робить `SUM() OVER()`

```sql
SELECT 
    id,
    user_id,
    amount,
    SUM(amount) OVER(PARTITION BY user_id) AS total_user
FROM orders;
```

`SUM()` — агрегатна функція.
`OVER()` перетворює її у **віконну функцію**.

Це означає, що:

* функція рахує значення **для групи рядків**
* **але не об’єднує рядки**, як `GROUP BY`

Тому кожен рядок залишається в результаті.

---

### `PARTITION BY`

```
PARTITION BY user_id
```

Розбиває таблицю на **групи (partition)** за `user_id`.

Для кожної групи SQL:

1. бере всі рядки з однаковим `user_id`
2. рахує `SUM(amount)`
3. записує результат у **кожен рядок цієї групи**

---

### Чим відрізняється від `GROUP BY`

`GROUP BY`

```sql
SELECT user_id, SUM(amount)
FROM orders
GROUP BY user_id;
```

→ повертає **один рядок на групу**

`SUM() OVER(PARTITION BY ...)`

→ залишає **усі рядки**, але додає обчислене значення.

---

### Якщо `PARTITION BY` не використовувати

```sql
SUM(amount) OVER()
```

SQL порахує суму **по всій таблиці** і запише її в кожний рядок.

---

# 4. ROW_NUMBER()

Нумерує рядки.

```sql
SELECT 
    id,
    user_id,
    amount,
    ROW_NUMBER() OVER(PARTITION BY user_id ORDER BY amount DESC) AS rank
FROM orders;
```

Результат

| user_id | amount | rank |
| ------- | ------ | ---- |
| 1       | 200    | 1    |
| 1       | 100    | 2    |
| 2       | 300    | 1    |
| 2       | 150    | 2    |

---

# 5. RANK()

Якщо є однакові значення — дає однаковий ранг.

```sql
SELECT 
    amount,
    RANK() OVER(ORDER BY amount DESC) AS rank
FROM orders;
```

---

# 6. DENSE_RANK()

Схоже на `RANK`, але **без пропусків**.

```
amount | rank
300 | 1
200 | 2
200 | 2
100 | 3
```

---

# 7. Running total (накопичувальна сума)

```sql
SELECT 
    id,
    amount,
    SUM(amount) OVER(ORDER BY id) AS running_total
FROM orders;
```

Результат

| id | amount | running_total |
| -- | ------ | ------------- |
| 1  | 100    | 100           |
| 2  | 200    | 300           |
| 3  | 150    | 450           |
| 4  | 300    | 750           |

---

# 8. LAG()

Дозволяє подивитися **попередній рядок**.

```sql
SELECT 
    id,
    amount,
    LAG(amount) OVER(ORDER BY id) AS previous_amount
FROM orders;
```

---

# 9. LEAD()

Дозволяє подивитися **наступний рядок**

```sql
SELECT 
    id,
    amount,
    LEAD(amount) OVER(ORDER BY id) AS next_amount
FROM orders;
```

---

# 10. Різниця з GROUP BY

### GROUP BY

```sql
SELECT user_id, SUM(amount)
FROM orders
GROUP BY user_id
```

Результат

| user_id | sum |
| ------- | --- |
| 1       | 300 |
| 2       | 450 |

**Рядки об'єднуються**

---

### Window function

```sql
SUM(amount) OVER(PARTITION BY user_id)
```

Рядки **не зникають**

---

# 11. Frame (рамка вікна)

Можна задати межі.

```sql
SUM(amount) OVER(
    ORDER BY id
    ROWS BETWEEN 1 PRECEDING AND CURRENT ROW
)
```

Означає:

**поточний рядок + попередній**

---

# 12. Найпопулярніші Window Functions

| функція    | що робить           |
| ---------- | ------------------- |
| ROW_NUMBER | номер рядка         |
| RANK       | ранг                |
| DENSE_RANK | ранг без пропусків  |
| SUM OVER   | накопичувальна сума |
| AVG OVER   | середнє             |
| LAG        | попередній рядок    |
| LEAD       | наступний рядок     |

---

✅ **Класичне інтерв'ю питання**

Знайти **найдорожче замовлення кожного користувача**

```sql
SELECT *
FROM (
    SELECT *,
           ROW_NUMBER() OVER(PARTITION BY user_id ORDER BY amount DESC) AS rn
    FROM orders
) t
WHERE rn = 1;
```
