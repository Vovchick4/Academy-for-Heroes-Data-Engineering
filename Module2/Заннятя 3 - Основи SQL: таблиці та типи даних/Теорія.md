# Урок 0: Створення бази даних — `CREATE DATABASE`

Перед тим як створювати таблиці, потрібно створити **базу даних**, у якій ці таблиці будуть зберігатися.

База даних — це контейнер, який містить:

* таблиці
* індекси
* views
* процедури
* функції

---

## 1. Команда CREATE DATABASE

Синтаксис:

```sql
CREATE DATABASE database_name;
```

Приклад:

```sql
CREATE DATABASE analytics_db;
```

Це створить нову базу даних з назвою **analytics_db**.

---

## 2. Перехід у базу даних

Після створення потрібно **вибрати базу**, в якій ми будемо працювати.

### PostgreSQL

```sql
\c analytics_db
```

### MySQL / MariaDB

```sql
USE analytics_db;
```

### SQL Server

```sql
USE analytics_db;
GO
```

---

## 3. Повний приклад

```sql
CREATE DATABASE analytics_db;

USE analytics_db;

CREATE TABLE users (
    userid INT AUTO_INCREMENT PRIMARY KEY,
    firstname VARCHAR(255),
    isactive BOOLEAN,
    createdat TIMESTAMP
);
```

---

## 4. Перевірка існування бази

У деяких системах можна написати:

```sql
CREATE DATABASE IF NOT EXISTS analytics_db;
```

Це корисно для **ETL pipelines**, щоб уникнути помилки, якщо база вже існує.

---

## 5. Видалення бази даних (DROP DATABASE)

⚠️ **Обережно:** ця команда видаляє базу повністю разом з усіма таблицями.

```sql
DROP DATABASE analytics_db;
```

---

## Як виглядає повний workflow

Інженер даних зазвичай робить так:

```
1️⃣ CREATE DATABASE
2️⃣ USE DATABASE
3️⃣ CREATE TABLE
4️⃣ INSERT DATA
5️⃣ SELECT DATA
```

## Урок 1: Основи SQL — Таблиці та Типи Даних

В інженерії даних ми не просто "зберігаємо" дані, ми проектуємо структури, які будуть ефективно обробляти мільйони рядків.

### 1. Що таке таблиця?

Таблиця — це об'єкт бази даних, який складається з **рядків** (записів) та **стовпців** (атрибутів). Кожен стовпець має чітко визначене ім'я та тип даних.

### 2. Створення таблиці (CREATE TABLE)

Синтаксис створення таблиці виглядає так:

```sql
CREATE TABLE Users (
    UserID int,
    FirstName varchar(255),
    IsActive boolean,
    CreatedAt timestamp
);

```

---

### 3. Основні типи даних

Для інженера даних критично важливо обирати правильний тип даних, щоб економити пам'ять та пришвидшувати запити.

| Категорія | Тип даних | Опис | Приклад |
| --- | --- | --- | --- |
| **Числа** | `INT` | Цілі числа | `1024` |
|  | `DECIMAL(p,s)` | Точні дробові числа (для грошей) | `99.99` |
|  | `FLOAT` / `REAL` | Наближені числа з комою | `3.1415` |
| **Текст** | `VARCHAR(n)` | Рядок змінної довжини | `'Data Pipeline'` |
|  | `TEXT` | Великий обсяг тексту | `'Long description...'` |
| **Дата/Час** | `DATE` | Тільки дата | `'2026-02-27'` |
|  | `TIMESTAMP` | Дата + час (часто з часовим поясом) | `'2026-02-27 15:30:00'` |
| **Логічні** | `BOOLEAN` | Істина або хиба | `TRUE` / `FALSE` |

---

### 4. Обмеження (Constraints)

Щоб дані були якісними (Data Quality), ми додаємо правила для стовпців:

* **`NOT NULL`**: Стовпець не може бути порожнім.
* **`UNIQUE`**: Всі значення в стовпці мають бути унікальними.
* **`PRIMARY KEY`**: Унікальний ідентифікатор рядка (комбінація `NOT NULL` та `UNIQUE`).
* **`DEFAULT`**: Значення за замовчуванням, якщо нічого не вказано.

> **Приклад "як у W3Schools":**
> Створимо таблицю для логів сенсорів (типова задача DE):
> ```sql
> CREATE TABLE sensor_readings (
>     reading_id SERIAL PRIMARY KEY,    -- Автоматичне число
>     sensor_name VARCHAR(50) NOT NULL, -- Назва сенсора
>     value FLOAT CHECK (value > 0),   -- Перевірка, щоб значення було > 0
>     recorded_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
> );
> 
> ```
> 
>

## Урок 2: Вибірка даних (SELECT) та унікальні значення (DISTINCT)

Після того, як ми створили таблицю, нам потрібно дістати з неї дані.

### 1. Оператор SELECT

Оператор `SELECT` використовується для вибору даних із бази даних.

**Синтаксис:**

```sql
SELECT column1, column2, ...
FROM table_name;

```

* Якщо ти хочеш вибрати **всі** стовпці, використовуй зірочку `*`:
`SELECT * FROM Orders;`
* Якщо потрібні лише конкретні дані (що краще для продуктивності):
`SELECT OrderID, CustomerName FROM Orders;`

---

### 2. Оператор SELECT DISTINCT

У таблицях часто зустрічаються дублікати (наприклад, один клієнт може зробити 10 замовлень). Оператор `DISTINCT` використовується для повернення лише **унікальних** (різних) значень.

**Синтаксис:**

```sql
SELECT DISTINCT column1, column2, ...
FROM table_name;

```

---

### 3. Приклад для інженера даних

Уявімо, що у нас є велика таблиця логів `UserActivity`, де записується кожна дія користувача на сайті.

**Таблиця `UserActivity`:**

| UserID | Action | Platform |
| --- | --- | --- |
| 101 | 'Login' | 'Android' |
| 102 | 'Login' | 'iOS' |
| 101 | 'Scroll' | 'Android' |
| 103 | 'Login' | 'Web' |
| 102 | 'Logout' | 'iOS' |

#### Сценарій А: Список усіх платформ

Якщо ми просто напишемо `SELECT Platform FROM UserActivity;`, ми отримаємо:
*Android, iOS, Android, Web, iOS*.

#### Сценарій Б: Список унікальних платформ

Щоб зрозуміти, з яких саме пристроїв до нас заходять люди, ми пишемо:

```sql
SELECT DISTINCT Platform FROM UserActivity;

```

**Результат:**

* Android
* iOS
* Web

---

### Важливі примітки (Tips)

* **Регістр:** SQL зазвичай не чутливий до регістру (case-insensitive), тому `SELECT` і `select` працюють однаково. Проте в професійному середовищі прийнято писати команди великими літерами для читабельності.
* **DISTINCT для кількох стовпців:** Якщо ти напишеш `SELECT DISTINCT City, Country`, SQL поверне унікальні **комбінації** міста та країни.

## Урок 3: Фільтрація даних — Оператор WHERE

Оператор `WHERE` використовується для витягування лише тих записів, які відповідають певній умові.

### 1. Синтаксис

```sql
SELECT column1, column2, ...
FROM table_name
WHERE condition;

```

> **Важливо:** `WHERE` можна використовувати не тільки в `SELECT`, а й у `UPDATE`, `DELETE` тощо.

---

### 2. Оператори в WHERE

Для фільтрації ми використовуємо стандартні математичні та спеціальні SQL-оператори:

| Оператор | Опис | Приклад |
| --- | --- | --- |
| `=` | Дорівнює | `WHERE City = 'Kyiv'` |
| `>` / `<` | Більше / Менше | `WHERE Price > 100` |
| `>=` / `<=` | Більше або дорівнює / Менше або дорівнює | `WHERE Age >= 18` |
| `<>` або `!=` | Не дорівнює | `WHERE Status <> 'Deleted'` |
| **`BETWEEN`** | Між певним діапазоном | `WHERE Date BETWEEN '2026-01-01' AND '2026-01-31'` |
| **`LIKE`** | Пошук за шаблоном | `WHERE Name LIKE 'Data%'` (починається на Data) |
| **`IN`** | Вибір з декількох значень | `WHERE Country IN ('Ukraine', 'Poland', 'USA')` |

---

### 3. Текстові vs Числові поля

SQL вимагає **одинарні лапки** для текстових значень. Числові поля пишуться без лапок.

* **Текст:** `SELECT * FROM Customers WHERE Country = 'Ukraine';`
* **Число:** `SELECT * FROM Products WHERE ProductID = 77;`

---

### 4. Комбінування умов (AND, OR, NOT)

Для складних запитів ми з'єднуємо умови:

* **`AND`**: Обидві умови мають бути правдивими.
* **`OR`**: Хоча б одна умова має бути правдивою.
* **`NOT`**: Умова НЕ має бути правдивою.

**Приклад (Data Engineering Style):**
Припустимо, нам треба знайти помилки (errors) в системі, які сталися на Android за останню добу:

```sql
SELECT * FROM SystemLogs
WHERE Status = 'Error' 
  AND Platform = 'Android'
  AND CreatedAt > '2026-02-26';

```

## Урок 4: Додавання даних — Оператор INSERT INTO

Оператор `INSERT INTO` використовується для вставки нових записів (рядків) у таблицю.

### 1. Два способи написання INSERT

**Спосіб А: Зі вказанням імен стовпців (Рекомендовано)**
Це найнадійніший метод, бо ти чітко кажеш, куди яке значення класти.

```sql
INSERT INTO table_name (column1, column2, column3, ...)
VALUES (value1, value2, value3, ...);

```

**Спосіб Б: Без вказання імен стовпців**
Ти можеш не писати назви стовпців, але тоді ти **зобов'язаний** передати значення для абсолютно всіх колонок у тому порядку, в якому вони були створені в таблиці.

```sql
INSERT INTO table_name
VALUES (value1, value2, value3, ...);

```

---

### 2. Практичний приклад

Давай наповнимо даними нашу таблицю `Users`, яку ми створили в першому уроці:

```sql
INSERT INTO Users (UserID, FirstName, IsActive, CreatedAt)
VALUES (1, 'Олексій', TRUE, '2026-03-02 10:00:00');

```

**Що важливо пам'ятати:**

* **Текст та Дати:** Завжди беруться в одинарні лапки (`'...'`).
* **Числа та Boolean:** Пишуться без лапок (`123`, `TRUE`, `FALSE`).
* **NULL:** Якщо стовпець дозволяє порожні значення, можна написати `NULL`.

---

### 3. Вставка кількох рядків за один раз (Bulk Insert)

В інженерії даних ми цінуємо продуктивність. Замість того, щоб виконувати 100 окремих запитів, краще зробити один "пакетний" запис:

```sql
INSERT INTO Users (UserID, FirstName, IsActive)
VALUES 
    (2, 'Марія', TRUE),
    (3, 'Дмитро', FALSE),
    (4, 'Анна', TRUE);

```

*Це працює набагато швидше, оскільки база даних відкриває транзакцію лише один раз.*

---

### 4. Робота з автоматичними полями (SERIAL / Identity)

Згадай наш приклад із сенсорами, де ми використовували `reading_id SERIAL`. Тип `SERIAL` сам генерує числа (1, 2, 3...). При вставці таких даних ми **пропускаємо** цей стовпець:

```sql
-- Ми не вказуємо reading_id, база сама його призначить
INSERT INTO sensor_readings (sensor_name, value)
VALUES ('Thermometer_01', 22.5);

```
